## 1. Introduction (Введение)

### 1.1 Purpose (Цель документа)  
Данный документ представляет собой спецификацию требований к разработке веб-сайта "Студопедия". В нем описываются требования к функционалу ресурса и пользовательским интерфейсам, пользователи, используемый набор технологий для разработки продукта, требования к продукту в контексте надежности и производительности, интерфейсы взаимодействия с продуктом, а также специфические ограничения, накладывающиеся на разработку продукта. Документ является основой для проектирования, разработки и тестирования системы.


### 1.2 Scope (Область применения)  
Проект направлен на создание образовательного портала - энциклопеди для студентов, преподавателей и авторов учебных материалов. Основные функции сервиса:  
- Просмотр и поиск учебных материалов: статей, лекций, пособий, по категориям.  
- Добавление, редактирование и удаление материалов авторизованными пользователями.  
- Оценка и комментирование материалов.  
- Личный кабинет с закладками и списком собственных публикаций.
- Заказ студенческой работы: платное написание домашней работы, курсовой работы, дипломной, реферата, доклада, эссе.
- Администрирование: управление пользователями, контентом, категориями, рекламой и статистикой.  

Целевая аудитория: студенты, аспиранты, преподаватели, авторы образовательного контента.

## 2. Overall Description (Общее описание)

### 2.1 Product functions (Функционал продукта)  
Система должна предоставлять:  
- регистрацию и авторизацию через email или социальные сети.  
- просмотр материалов с фильтрацией по категориям и тегам.
- полнотекстовый поиск.
- добавление новых материалов с возможностью прикрепления файлов, изображений). 
- редактирование и удаление собственных материалов.
- комментирование и оценку (лайки, дизлайки, рейтинг). 
- добавление материалов в закладки.  
- личный кабинет (профиль, закладки, мои материалы, настройки).  
- административную панель для управления пользователями, контентом, категориями, рекламой и статистикой.
- систему уведомлений о комментариях, статусе модерации.  

### 2.2 User characteristics (Описание пользователей)  
- **Гость**: просматривает общедоступные материалы, выполняет поиск, может зарегистрироваться.  
- **Зарегистрированный пользователь (читатель)**: дополнительно может комментировать, оценивать, добавлять материалы в закладки.  
- **Автор (пользователь с правом публикации)**: может добавлять, редактировать и удалять свои материалы после модерации.  
- **Модератор**: проверяет новые материалы и комментарии.  
- **Администратор**: полный доступ к управлению системой.

### 2.3 Assumptions and dependencies (Влияющие факторы и зависимости)  
- Наличие стабильного интернет-соединения у пользователей.  
- Использование современных браузеров (Chrome, Firefox, Edge, Safari).  
- Серверная инфраструктура должна обеспечивать круглосуточную доступность и резервное копирование.
- Наличие базы данных для хранения пользовательской информации, материалов и комментариев.
- Корректную работу системы электронной почты для отправки уведомлений и восстановления пароля.

### 2.4 Constraints (Ограничения)  
- Соблюдение законодательства в области защиты персональных данных пользователей.
- Размещаемый контент не должен нарушать законодательство.
- Веб-приложение должно работать по протоколу HTTPS.
- Интерфейс должен быть адаптивным (мобильные устройства, планшеты, ПК).

### 6.1 Акторы и прецеденты  
**Акторы:**  
- Гость  
- Зарегистрированный пользователь (читатель)  
- Автор (расширение прав читателя)  
- Модератор  
- Администратор  

**Прецеденты (UC):**  
- UC-01: Регистрация  
- UC-02: Авторизация  
- UC-03: Просмотр материалов  
- UC-04: Поиск материалов  
- UC-05: Добавление материала  
- UC-06: Редактирование материала  
- UC-07: Удаление материала  
- UC-08: Комментирование  
- UC-09: Оценка материала  
- UC-10: Управление закладками  
- UC-11: Модерация материалов  
- UC-12: Управление пользователями  
- UC-13: Управление категориями  
- UC-14: Просмотр статистики  

**Связи:**  
- UC-05 включает UC-03? (нет)  
- UC-08 может расширять UC-03 (опционально)  
- UC-11 является специализацией для модератора.  

*Диаграмма UseCase (текстовое описание):*  
Акторы связаны с соответствующими прецедентами. Гость взаимодействует только с UC-01, UC-02, UC-03, UC-04. Зарегистрированный пользователь дополнительно с UC-08, UC-09, UC-10. Автор – с UC-05, UC-06, UC-07. Модератор – с UC-11. Администратор – с UC-11, UC-12, UC-13, UC-14.

### 6.2 Детальные сценарии ключевых прецедентов  

#### **Прецедент UC-05: Добавление материала**  
| Элемент            | Описание                                                                 |
|--------------------|--------------------------------------------------------------------------|
| Актор              | Автор                                                                    |
| Целевой уровень    | Уровень моря                                                             |
| Предусловия        | Пользователь авторизован и имеет роль «Автор». Находится на странице «Добавить материал». |

**Главный успешный сценарий:**  
1. Автор нажимает кнопку «Добавить материал».  
2. Система отображает форму с полями: заголовок, категория, текст (WYSIWYG-редактор), теги, прикреплённые изображения/файлы.  
3. Автор заполняет обязательные поля и при необходимости загружает файлы.  
4. Автор нажимает «Предварительный просмотр».  
5. Система отображает предварительную версию материала.  
6. Автор подтверждает публикацию, нажимая «Опубликовать».  
7. Система сохраняет материал со статусом «на модерации» (если требуется) или «опубликован».  
8. Система уведомляет автора об успешной отправке.  

**Расширения:**  
- *3a. Не заполнены обязательные поля*: система подсвечивает ошибки, автор возвращается к шагу 3.  
- *4a. Некорректный формат файла*: система сообщает об ошибке, автор заменяет файл.  
- *7a. Ошибка соединения*: система сохраняет черновик локально или предлагает повторить позже.  

#### **Прецедент UC-11: Модерация материалов**  
| Элемент            | Описание                                                                 |
|--------------------|--------------------------------------------------------------------------|
| Актор              | Модератор                                                                |
| Целевой уровень    | Уровень моря                                                             |
| Предусловия        | Модератор авторизован, есть материалы, ожидающие проверки.              |

**Главный успешный сценарий:**  
1. Модератор открывает раздел «Модерация».  
2. Система отображает список материалов со статусом «на модерации».  
3. Модератор выбирает материал для проверки.  
4. Система показывает полный материал и комментарии автора.  
5. Модератор проверяет материал на соответствие правилам.  
6. Модератор принимает решение: опубликовать, отклонить (с указанием причины) или отправить на доработку.  
7. Система обновляет статус материала и отправляет уведомление автору.  
8. Модератор возвращается к списку (шаг 2).  

**Расширения:**  
- *5a. Материал содержит запрещённый контент*: модератор отклоняет с причиной, система блокирует возможность повторной публикации.  

#### **Прецедент UC-12: Управление пользователями**  
| Элемент            | Описание                                                                 |
|--------------------|--------------------------------------------------------------------------|
| Актор              | Администратор                                                            |
| Целевой уровень    | Уровень моря                                                             |
| Предусловия        | Администратор авторизован.                                               |

**Главный успешный сценарий:**  
1. Администратор открывает раздел «Пользователи».  
2. Система отображает список пользователей с фильтрацией и поиском.  
3. Администратор выбирает пользователя.  
4. Система показывает детальную информацию.  
5. Администратор может заблокировать/разблокировать, удалить, изменить роль.  
6. Система подтверждает действие и обновляет данные.  

**Расширения:**  
- *5a. Удаление пользователя*: система запрашивает подтверждение, выполняет каскадное удаление или деактивацию.
